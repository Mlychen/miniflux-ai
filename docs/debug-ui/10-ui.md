# 10. Debug UI 功能与接口对接

本页面描述 Debug UI 的最小功能、页面结构、调用的后端接口与推荐交互。

目标：你打开 `http://IP:PORT/debug/` 后，不需要任何额外工具即可完成常见调试动作。

## 页面结构（建议）

### 区域 A：入口与连接状态

显示：
- 当前页面 origin（例如 `http://192.168.1.10:80`）
- API 基础前缀（固定 `/miniflux-ai`）
- 最后一次请求耗时、状态码

### 区域 B：手动触发处理

控件：
- `entry_id` 输入框（支持粘贴）
- 按钮：“触发处理”（POST）

请求：
- `POST /miniflux-ai/manual-process`
- body：`{"entry_id": 123}`、`{"entry_id":"123"}`，或 `{"entry_id":"123","trace_id":"32位hex"}`（trace_id 为本次处理链路 ID，可选）

展示：
- 成功返回 JSON
- 失败返回 JSON + HTTP 状态码
- 成功时显示 `trace_id`（处理链路 ID，用于在“处理追踪”中定位本次处理）并可一键带入”处理追踪”查询框

### 区域 C：请求池指标（metrics）

控件：
- 按钮：“刷新指标”
- （可选）自动刷新开关（例如每 10 秒）

请求：
- `GET /miniflux-ai/user/llm-pool/metrics`

展示：
- `total_calls`
- `total_errors`
- `total_rejected`

### 区域 D：失败条目列表（failed-entries）

控件：
- limit 输入框（默认 100）
- 按钮：“加载失败条目”

请求：
- `GET /miniflux-ai/user/llm-pool/failed-entries?limit=100`

展示（表格）：
- `task_id`
- `canonical_id`
- `status`（通常为 `retryable` / `dead`）
- `attempts` / `max_attempts`
- `last_error` / `error_key`
- `next_retry_at`
- `created_at` / `updated_at`
- `url`（若存在，点击打开）
- `title`（若存在）
- 操作：重入队（对单条 task）

重入队请求：
- `POST /miniflux-ai/user/llm-pool/clear`
- body：`{"task_id":"..."}` 或 `{"task_id":123}`

### 区域 E：清空池（危险操作）

控件：
- 按钮：“清空池”
- 二次确认（浏览器 confirm 或输入确认词）

请求：
- `POST /miniflux-ai/user/llm-pool/clear`
- body：`{}`

展示：
- 批量重入队成功后自动刷新 metrics 与 failed-entries

### 区域 F：任务排障（failure groups）

控件：
- `status` 筛选（`retryable|dead`）
- `error/error_key` 输入（支持模糊输入，后端会归一化）
- 分组 `limit`
- 按钮：“加载分组”
- 按钮：“按筛选重入队”（基于当前 `status/error/error_key`）

请求：
- `GET /miniflux-ai/user/tasks/failure-groups?status=&error=&error_key=&limit=&offset=`

展示（分组表格）：
- `status`
- `error_key`
- `example_error`
- `count`
- `latest_updated_at`
- `oldest_created_at`
- 操作：
  - “查看任务”：加载该分组任务样本
  - “重入队该组”：对该分组批量重入队

样本请求：
- `GET /miniflux-ai/user/tasks/failure-groups/tasks?status=&error=&error_key=&limit=&offset=&include_payload=`

样本展示：
- `id`
- `status`
- `attempts/max_attempts`
- `error_key`
- `trace_id`（处理链路 ID）
- `updated_at`
- 操作：`POST /miniflux-ai/user/tasks/<task_id>/requeue`（单任务重入队）

任务详情：
- 输入 `task_id` 后调用：
  - `GET /miniflux-ai/user/tasks/<task_id>`
- 用于查看完整任务字段（`attempts/max_attempts/last_error/error_key/payload` 等）。

## 推荐的 UI 交互规则

- 所有请求都带请求耗时统计（`performance.now()` 前后差）
- 对非 2xx 响应展示：
  - HTTP 状态码
  - 响应体（若为 JSON）
- 展示“可复制”的原始 JSON（textarea 只读即可）
- 对危险操作必须二次确认（清空池）

## 并行开发建议

UI 可以先不依赖后端：
- 用 mock JSON（写在页面里）把表格渲染与交互做好
- 等入口反代与后端接口就绪后，再把 mock 替换成 `fetch`

## 后续增强（可选）

- 最近手动触发历史（entry_id + 时间 + 结果）保存在 localStorage
- 一键复制请求 curl（便于复现）
- 显示 miniflux-ai 版本号与启动模式（需要后端新增一个 `/miniflux-ai/debug/info`，当前未实现）
