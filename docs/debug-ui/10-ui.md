# 10. Debug UI 功能与接口对接

本页面描述 Debug UI 的最小功能、页面结构、调用的后端接口与推荐交互。

目标：你打开 `http://IP:PORT/debug/` 后，不需要任何额外工具即可完成常见调试动作。

## 页面结构（建议）

### 区域 A：入口与连接状态

显示：
- 当前页面 origin（例如 `http://192.168.1.10:80`）
- API 基础前缀（固定 `/miniflux-ai`）
- 最后一次请求耗时、状态码

### 区域 B：手动触发处理

控件：
- `entry_id` 输入框（支持粘贴）
- 按钮：“触发处理”（POST）

请求：
- `POST /miniflux-ai/manual-process`
- body：`{"entry_id": 123}` 或 `{"entry_id":"123"}`

展示：
- 成功返回 JSON
- 失败返回 JSON + HTTP 状态码

### 区域 C：请求池指标（metrics）

控件：
- 按钮：“刷新指标”
- （可选）自动刷新开关（例如每 10 秒）

请求：
- `GET /miniflux-ai/user/llm-pool/metrics`

展示：
- `total_calls`
- `total_errors`
- `total_rejected`

### 区域 D：失败条目列表（failed-entries）

控件：
- limit 输入框（默认 100）
- 按钮：“加载失败条目”

请求：
- `GET /miniflux-ai/user/llm-pool/failed-entries?limit=100`

展示（表格）：
- `entry_key`
- `status`（failed/expired/dropped）
- `attempts_used` / `max_attempts`
- `created_at` / `last_attempt_at` / `ttl_seconds`
- `url`（若存在，点击打开）
- 操作：重置（对单条 entry_key）

重置请求：
- `POST /miniflux-ai/user/llm-pool/clear`
- body：`{"entry_key":"..."}`

### 区域 E：清空池（危险操作）

控件：
- 按钮：“清空池”
- 二次确认（浏览器 confirm 或输入确认词）

请求：
- `POST /miniflux-ai/user/llm-pool/clear`
- body：`{}`

展示：
- 清空成功后自动刷新 metrics 与 failed-entries

## 推荐的 UI 交互规则

- 所有请求都带请求耗时统计（`performance.now()` 前后差）
- 对非 2xx 响应展示：
  - HTTP 状态码
  - 响应体（若为 JSON）
- 展示“可复制”的原始 JSON（textarea 只读即可）
- 对危险操作必须二次确认（清空池）

## 并行开发建议

UI 可以先不依赖后端：
- 用 mock JSON（写在页面里）把表格渲染与交互做好
- 等入口反代与后端接口就绪后，再把 mock 替换成 `fetch`

## 后续增强（可选）

- 最近手动触发历史（entry_id + 时间 + 结果）保存在 localStorage
- 一键复制请求 curl（便于复现）
- 显示 miniflux-ai 版本号与启动模式（需要后端新增一个 `/miniflux-ai/debug/info`）
